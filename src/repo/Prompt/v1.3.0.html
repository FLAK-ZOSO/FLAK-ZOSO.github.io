v1.3.0-beta Release notes 31/1/2022

Boolean exception handling for commands.loop, commands.run and source.run

Empty string returned by perform.main when a exceptions.CommandException is triggered

Full error catching in command run for invalid inputs

Help .txt files updated for echo, cls and loop

Added int argument to command cls, that specifies the number of newlines (\n) to print

v1.3.0 Release notes 1/2/2022

Fixed bug in parsing.py due to missing backslash (\\) between path and file

You can find the complete changelog here

v1.3.1 Release notes 1/2/2022

Improved command help with KeyError handling and check for keys in synonims dictionary

Removed a bug due to KeyError catched before searching for aliases of quit command in perform.main

Removed dangerous and useless pointer to commands.commands dictionary in perform.py

Added help .txt file for complex command directory

You can find the complete changelog here

v1.3.2 Release notes 3/2/2022

Improved efficiency in command close by assigning variables.getCurrentPath() and variables.getDefaultPath() to variables to avoid new calls during the execution

Inline argument check for command setVar

Full error catching in command makeFile for invalid inputs

Now also relative paths are capitalized, thanks to a .capitalize() called in parsing.path

close command is now handled as a normal command, and it takes a optional (and useless) positional argument to match perform.py's call

You can find the complete changelog here

The code was written with Python 3.10.2, it can be used with Python 3.10+ without bugs
