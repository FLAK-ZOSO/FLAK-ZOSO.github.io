v1.2.0 Release notes 30/1/2022

Inline comments are now allowed with //, in source files too

Multiline comments are now allowed with /* opening and *\ closing

off and on are now aliases to false and true in setVar command

exceptions.CommandException only prints the first not recognized word instead of the full input

You can find the complete changelog here

v1.2.1 Release notes 31/1/2022

Command echo now reduces the spaces. echo Hello, World! -> Hello, World!

Function checkComments and global variable comment are now stored in comments.py

Fixed bug in checkComments: now illimited comments per line are allowed

You can find the complete changelog here

v1.2.2 Release notes 31/1/2022

Fixed bug in comments.py, inserting *\ without a previous opening tag raised a RecursionError after printing around 989 times the [WARNING]

Full error catching in command setVar for invalid inputs

Full error catching in commands move, makeDir and makeSource for invalid inputs

You can find the complete changelog here

v1.2.3 Release notes 31/1/2022

Fixed issue #26 with pull request #27: relative path usage is now enabled via os.chdir()

You can find the complete changelog here

v1.2.4 Release notes 31/1/2022

Full PermissionError catching in commands makeFile, makeSource and makeDirectory

Function makeFile now returns False when a PermissionError is raised, otherwise True

You can find the complete changelog here
